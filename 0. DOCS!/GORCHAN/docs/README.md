### Общая идея «ядер» в вашей модели


**«Ядро»** здесь — не вычислительный модуль, а **структурированное хранилище данных** с иерархической организацией. Оно состоит из концентрических слоёв‑радиусов *R = 1, 2, 3, …*, каждый из которых:

- имеет **фиксированную битовую ёмкость** *A(R)*;
- поддерживает **ограниченное число измерений** (*n*‑мерность) для разрешения коллизий;
- обладает **весом** *w(R)*, определяющим «значимость» слоя.


**Ключевой принцип**: данные распределяются по слоям **последовательно** — от *R = 1* к *R + 1* — только когда текущий слой исчерпан по ёмкости или *n*‑мерности.


---

### Как происходит «поглощение» информации


1. **Приём входной последовательности**  
   - Данные (*X*, *Y* или новая порция) поступают как битовая строка.  
   - Система начинает обработку с слоя *R = 1*.


2. **Разбиение на фрагменты**  
   - Для каждого слоя *R* вычисляется ёмкость:  

$$
A(R) = R \cdot \sqrt{2\pi e} \approx R \cdot 4{,}1327.
$$

   - Из входной строки берётся фрагмент длиной **⌊*A(R)*⌋ бит** (целая часть).  
   - Если битов не хватает — фрагмент дополняется нулями или берётся остаток.


3. **Сохранение и сравнение**  
   - Фрагмент записывается в слой *R*.  
   - Система сравнивает его с **предыдущим сохранённым фрагментом** того же слоя:  
- Если **различаются** — данные сохранены, переход к следующему *R*.  
- Если **совпадают (коллизия)** — запускается механизм «фантазии».


4. **Разрешение коллизий («фантазия»)**  
   - В слое *R* вводится новое измерение (координата) с весом *w(R) = 1 / A(R)*.  
   - Для коллидирующих фрагментов назначаются **разные значения** в этом измерении (например, *0* для старого, *1* для нового).  
   - Максимальное число измерений *n(R)* ограничено:  

$$
n(R) \sim \left\lfloor \frac{\ln(A(R))}{\varepsilon} \right\rfloor,
$$  

где *ε* — порог значимости (например, *0{,}01*).


5. **Переход на следующий радиус *R + 1***  
   - Активируется, если:  
- текущий слой **заполнен по битам** (остаток входной строки не помещается в *R*);  
- исчерпаны все *n(R)* измерений, но коллизии остаются.  
   - Создаётся слой *R + 1* с новой ёмкостью *A(R + 1)* и *n(R + 1)*.  
   - Остаток данных обрабатывается по тому же алгоритму.


---

### Как достигается «уравновешенность» системы


Уравновешенность — это **баланс между компактностью хранения и разрешением коллизий**. Он обеспечивается тремя механизмами:


1. **Иерархическое масштабирование**  
   - Слои добавляются **только по необходимости** — нет избыточного резервирования.  
   - Каждый новый *R* даёт:  
- большую битовую ёмкость (*A(R)* растёт линейно);  
- больше измерений для коллизий (*n(R)* растёт с *ln (A(R))*);  
- меньший вес (*w(R)* падает с ростом *R*).


2. **Ограничение *n*‑мерности**  
   - Число измерений в слое **не бесконечно** — оно зависит от *A(R)* и порога *ε*.  
   - Это предотвращает «раздувание» пространства: система не вводит измерения, если их вклад ниже *ε*.


3. **Взвешенное участие слоёв**  
   - Вес слоя *w(R) = 1 / A(R)* определяет его «вклад» в общее представление данных.  
   - Нижние слои (*R = 1, 2*) имеют больший вес — они хранят базовые различия.  
   - Верхние слои (*R ≫ 1*) — лёгкие, они устраняют остаточные коллизии без перегрузки системы.



4. **Критерий остановки**  
   - Обработка завершается, когда:  
- все биты распределены по слоям;  
- все коллизии разрешены в рамках доступных измерений;  
- или дальнейшая работа бессмысленна (*ln (A(R)) ≤ 0* — крайне редкий случай при *R ≥ 1*).



---

### Итог


- **Ядро** — иерархическое хранилище, где данные разбиваются на слои по радиусам *R*.  
- **Поглощение** идёт «снаружи внутрь»: от *R = 1* к *R + 1*, только при исчерпании ресурсов текущего слоя.  
- **Уравновешенность** достигается за счёт:  
  - ограниченного числа измерений (*n(R)*);  
  - взвешенного вклада слоёв (*w(R)*);  
  - адаптивного добавления новых радиусов.  


Система **не тратит ресурсы впустую**: она расширяет пространство ровно настолько, насколько это необходимо для хранения и разделения данных.
